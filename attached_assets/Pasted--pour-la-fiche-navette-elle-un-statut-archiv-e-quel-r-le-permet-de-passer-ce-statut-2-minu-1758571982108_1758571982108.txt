Decided on archiving roles (5 seconds)
Excellente question ! Après avoir analysé le système de permissions, voici quels rôles peuvent passer une fiche navette au statut 'archivée' :
🔒 Rôles autorisés à archiver :
1. ADMIN
* ✅ Peut archiver depuis n'importe quel état
* ✅ Accès complet sans restriction
2. CD (Conseil Départemental)
* ✅ Peut archiver depuis : SUBMITTED_TO_CD
* ❌ Ne peut pas archiver depuis les autres états
3. RELATIONS_EVS
* ✅ Peut archiver depuis :
    * ACCEPTED_EVS (si l'EVS a accepté)
    * CLOSED (fiche fermée normalement)
* ❌ Ne peut pas archiver depuis les autres états
❌ Rôles qui NE peuvent PAS archiver :
* EMETTEUR : Pas d'accès archivage
* EVS_CS : Pas d'accès archivage
* SUIVI_PROJETS : Pas d'accès archivage
🎯 Logique métier :
L'archivage suit la logique du workflow :
* CD : Archive les fiches refusées/non validées
* RELATIONS_EVS : Archive après acceptation EVS ou clôture normale
* ADMIN : Contrôle total pour cas exceptionnels
💡 En pratique :
Si une fiche est archivée, c'est qu'un utilisateur avec l'un de ces 3 rôles (ADMIN, CD, ou RELATIONS_EVS) l'a fait depuis un état autorisé selon les règles de transition.
L'ADMIN a le pouvoir le plus large pour l'archivage ! 🔧
