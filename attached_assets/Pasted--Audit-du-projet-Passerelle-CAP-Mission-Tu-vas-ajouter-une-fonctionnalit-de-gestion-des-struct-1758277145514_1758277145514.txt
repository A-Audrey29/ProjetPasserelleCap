# Audit du projet Passerelle CAP

## Mission
Tu vas ajouter une fonctionnalité de gestion des structures à ce projet existant. Avant de coder quoi que ce soit, tu DOIS comprendre comment fonctionne le projet pour t'y intégrer correctement.

## Étape 1 : Découverte de l'architecture

**Explore le projet et dis-moi :**
- Comment est organisé le code ? (quels dossiers principaux ?)
- Quelles technologies sont utilisées ? (framework, base de données, etc.)
- Comment fonctionne l'interface d'administration actuelle ?

## Étape 2 : Analyse de la base de données

**Examine la structure de données :**
- Quel ORM/système de base de données est utilisé ?
- Trouve et montre-moi les tables existantes
- Y a-t-il déjà des données sur les structures/organisations ?
- Comment sont définis les schémas de base de données ?

## Étape 3 : Patterns du code existant

**Comprends les conventions :**
- Comment sont construits les formulaires dans ce projet ?
- Comment sont gérées les listes/tableaux avec des données ?
- Comment fonctionnent les routes API ?
- Quel système de styles CSS est utilisé ?
- Comment sont gérées les validations (côté client et serveur) ?

## Étape 4 : Interface d'administration

**Analyse l'admin existant :**
- Fais une capture d'écran de l'interface d'administration
- Comment sont organisés les onglets/menus ?
- Comment ajouter un nouvel onglet dans cette interface ?
- Comment fonctionne l'authentification admin ?

## Étape 5 : Gestion d'état et API

**Comprends le flux de données :**
- Comment sont structurées les requêtes API (endpoints, méthodes) ?
- Comment est géré l'état côté client (store, cache) ?
- Y a-t-il des middlewares ou des hooks personnalisés ?
- Comment sont gérées les erreurs et les notifications ?

## Étape 6 : Contraintes et objectif final

**Pour info, voici ce qu'on va créer :**
- Un onglet "Structures" dans l'admin
- Liste des structures (tableau avec tri/filtre/recherche)
- Formulaire pour créer/modifier des structures
- Import de données depuis un fichier CSV
- Champs requis : nom, adresse, ville, EPCI, nom du contact, email du contact, type de structure, activités

## Résultat attendu

Un rapport détaillé qui me montre que tu comprends :
1. L'architecture du projet
2. Les conventions de code
3. Comment s'intégrer sans casser l'existant
4. Les noms exacts des fichiers/dossiers/tables importants
5. Comment implémenter le système de gestion des structures en respectant ces conventions

## Important
Ne modifie RIEN pour l'instant. Je veux d'abord être sûr que tu comprends le projet avant qu'on ajoute la gestion des structures.
